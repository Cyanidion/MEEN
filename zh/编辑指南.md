欢迎参与**自新博物馆世设维基**的编纂工作。本维基用于系统化记录架空世界设定，并为**协作创作**、**资料整理**、**公开展示**与 **TRPG 游戏**提供稳定的文本基础。

<blockquote class="custom-blockquote-warning">
若您计划在本维基添加、修改、搬运、扩写或重构任何内容，请务必先阅读并遵守本指南。
</blockquote>

## 说明与适用范围

本文档是本维基的**编辑指南**，主要规定页面创建、路径命名、Markdown 语法、页面结构、模板使用、特殊语法、资产管理与发布检查等基本要求，以维护维基结构与内容风格的一致性。

本指南只处理**页面编辑与格式问题**。若您计划新增设定、修改既有设定、参与互评、处理争议或推进时间线，请同时阅读[**共创指南**](/共创指南)。

如有疑问或建议，可在页面讨论区留言，或前往[**项目协作群**](https://qm.qq.com/q/F0m6zn9UU8)交流。

## 开始编辑

### 了解维基

编辑前，请先理解对应世界观的主要基调与基准设定。自新博物馆世设维基不是单纯的信息堆放处，而是面向多个架空世界的协作资料库。每个页面都应服务于其所属世界的整体叙事、结构与可读性。

### 编辑方式

在已有页面右上角点击**编辑**即可进入编辑界面。创建新页面时，请使用顶部的**创建页面**按钮，并按照本指南的[**页面命名与路径**](#页面命名与路径)规范填写路径。

除非您明确知道自己正在处理原始 HTML 页面，否则请优先使用 **Markdown 编辑器**。Markdown 语法简洁、便于审阅，也更适合多人协作。

原始 HTML 编辑器主要用于迁移旧页面或处理特殊布局，不建议作为常规写作方式。编辑完成后点击保存即可，页面历史版本会由系统自动保留。

## 基础语法

### Markdown

推荐使用 **Markdown** 作为主要编辑语言。Markdown 适合编写标题、段落、列表、链接、表格、引用与代码块，能够在保持文本清晰的同时降低排版成本。

您可在 Markdown 编辑器左下角点击问号图标查看快捷参考，也可参阅 [**Wiki.js 官方 Markdown 指南**](https://docs.requarks.io/editors/markdown)。

### HTML

Markdown 中可以嵌入少量 HTML，以实现复杂布局或调用本站已配置的特殊样式，例如折叠区块、提示框或复杂表格。

请勿在普通页面中随意加入 `<style>`、`<script>` 或大量内联样式。全站可复用的样式与脚本应由管理者统一维护；单页临时样式容易破坏维基风格，也会增加后续维护成本。

如需新增可复用的视觉样式、页面组件或模板语法，请先在协作群中讨论，再交由管理者统一加入全站 CSS 或 HTML 注入。

## 页面命名与路径

页面路径承担分类功能，应采用清晰、稳定的树状结构。基本格式为：

`/世界名/分类/条目名`

常见示例：

| 类型 | 推荐路径 |
| --- | --- |
| 人物 | `/弥恩诸世/人物汇总/人物名` |
| 地点 | `/弥恩诸世/人文地理/地点/地点名` |
| 组织 | `/弥恩诸世/政权组织/组织/组织名` |
| 事件 | `/弥恩诸世/历史沿革/事件名` |
| 神明 | `/弥恩诸世/世界总览/信仰体系/神明名` |

路径应以页面的长期归属为依据，而不是以一时用途或当前编辑者习惯为依据。请避免在根目录创建大量独立页面，也不要使用过深嵌套。一般情况下，路径层级建议不超过三级；若确需更深层级，应确认该分类具有长期扩展价值。

页面名称应简明、准确，尽量使用设定中的正式名称。若存在同名或近似名称，应建立**消歧义页**，而不是用随意后缀区分。

## 页面结构与行文

### 页面结构

同一子分类下的页面应遵循统一的基础结构，以确保信息完整、风格一致。若某个页面被标记为**模板参考**，编辑同类页面时应优先参照该页面。

页面正文不必在开头重复当前页面标题，因为页面标题已经显示于页面上方。正文中建议使用二级标题组织主要章节，再使用三级标题展开细节，以避免页面索引过重。

### 行文规范

页面文字应自然、流畅、准确。叙述语气应符合所属世界的基调，同时保持维基文本应有的清晰性。除非页面本身是世界内文献、角色独白或跑团记录，一般不宜使用过于口语化、情绪化或立场过强的表达。

基本规范如下：

- 中文、英文、数字与单位之间建议空一格，如 `DND 3.5E`、`221 纪年`、`10 gp`、`DC 20`。
- 段落之间应空一行，以形成清晰间距。
- 已确定的专有名词应保持一致，不要随意创造同义写法。
- 概念首次出现时可添加内链；同一页面内反复出现的同一概念，不必每次都链接。
- 编辑意图可用 HTML 注释说明，如 `<!-- 此处等待东部地区设定补充后再修订 -->`。

## 链接、术语与标签

### 内部链接

内部链接应服务于阅读，而不是为了堆砌索引。通常情况下，重要概念、人物、地区、政权、历史事件在正文首次出现时添加链接即可。

推荐写法：

- `[弥恩诸世](/弥恩诸世/世界总览)`
- `[帝国监护政府](/弥恩诸世/政权组织/政权/帝国监护政府)`

不推荐写法：

- `[点击这里](/弥恩诸世/世界总览)`

链接文字应直接说明目标页面内容，避免使用“这里”“这个页面”等含糊表达。

若目标页面尚未创建，可以先保留普通文本；若确有必要提前建立页面，应使用**占位**模板标明状态。

### 术语提示

术语提示用于在不打断正文阅读的情况下解释概念，适合用于世界观专有名词、规则术语、组织简称或读者可能不熟悉的概念。

术语提示使用如下写法，需注意，术语提示功能依赖于 JS 代码实现，因此被`{{}}`框定的内容不应使用 Markdown 标记语言，否则可能造成引用失效：

`{{术语名|/页面路径}}`

示例：

`弥恩的奥术体系依赖{{魔网|/弥恩诸世/世界总览/概念/魔网}}运行。`

术语提示会尝试检索目标页面中与“术语名”一致的标题，并显示该标题下的内容直至下一个同级或更高级的标题。点击术语后，会跳转至对应页面的相关段落。

术语提示应适度使用。若一个概念对理解当前段落至关重要，优先使用普通内链；若只是补充说明，则可以使用术语提示。

### 分类与标签

在你创建一个页面时，应该为其设置适应的**标签**。

页面路径承担“分类”功能，而标签承担“横向关联”功能，二者不应混淆。

**分类**由页面路径决定，主要用于内容归档和层级导航，例如 `/弥恩诸世/人物汇总/人物名` 或 `/弥恩诸世/政权组织/组织/组织名`。

**标签**是对页面内容的关键词描述，主要用于跨分类检索、主题聚合与关联发现。一个页面可以拥有多个标签。

标签应遵循**具体、通用、稳定**的原则。在给页面设置标签前，你应该浏览以下标签，并从中选取符合需求的标签。你必须使用**必须项**作为标签。

如此处没有适应的标签，请遵照{{引入内容|/共创指南}}的方式进行创建。

可使用的标签：

| 类型 | 必须项 |可选项|
| --- | --- | --- |
| 导航汇总 | `导航`/`汇总` | - |
| 需求编辑 | `待编辑` | - |
| 信仰体系 | `信仰` | `<神名>` |
| 人文地理 | `自然`/`人文`、`<东南西北中>部` | `经济`、`语言` |
| 历史关联 | `历史` | `<发生地>` |
| 政权组织 | `政权`/`组织`、`<东南西北中>部` | `<所属政权>` |
| 人物关联 | `人物`、`<东南西北中>部`、`<国籍>/无国籍` | `<组织名>`、`玩家角色`、`重要人物`、`皇室` |
| 意识形态 | `意识形态` | - |
| 增改规则 | `规则`、`<所属路径>` | - |

每个页面的标签建议控制在 **3 至 8** 个之间。标签应与页面主旨高度相关，不应因为某个概念在页面中被顺带提及，就为其添加标签。

## 页面模板

页面模板用于提醒读者和编辑者：该页面目前是否稳定、是否可引用、是否需要讨论，或属于何种内容性质。

模板通常置于页面正文开头。若一个页面同时需要多个模板，应控制数量，优先保留最能说明页面状态的模板。例如**跑团记录**与**剧透警告**可以并用，但**草稿**、**修订中**、**确认中**通常不应同时出现。

基本格式：

- `【【模板名】】`
- `【【模板名|编辑者签名】】`
- `【【模板名|编辑者签名|补充说明】】`

示例：

`【【草稿|罗莎|今天吃了草莓芭菲】】`

### 共创流程模板

| 模板 | 用途 | 示例 |
| --- | --- | --- |
| 提案 | 初步构想，尚未进入正式草稿 | `【【提案】】` |
| 草稿 | 内容正在形成，允许存在待定项 | `【【草稿】】` |
| 待互评 | 初稿已完成，等待他人审阅 | `【【待互评】】` |
| 修订中 | 正在根据意见修改 | `【【修订中|罗莎】】` |
| 确认中 | 已进入确认期 | `【【确认中】】` |
| 已锚定 | 内容已经稳定，可作为正式设定引用 | `【【已锚定|自新博物馆编录组】】` |

### 设定层级模板

| 模板 | 用途 | 示例 |
| --- | --- | --- |
| 核心设定 | 世界根基内容，不应由个人单独修改 | `【【核心设定】】` |
| 重要设定 | 会影响多个页面或多个创作者的内容 | `【【重要设定】】` |
| 普通设定 | 局部人物、地点、事件或传说等内容 | `【【普通设定】】` |
| 衍生设定 | 跑团记录、非正史、平行宇宙、废案等内容 | `【【衍生设定】】` |

### 维护与性质模板

| 模板 | 用途 | 示例 |
| --- | --- | --- |
| 需要改进 | 内容质量或完整度不足 | `【【需要改进】】` |
| 占位 | 页面仅用于预留条目 | `【【占位】】` |
| 模板参考 | 可供同类页面参照 | `【【模板参考】】` |
| 精选 | 高质量参考页面 | `【【精选】】` |
| 消歧义 | 用于区分同名或近似条目 | `【【消歧义】】` |
| 剧透警告 | 包含关键剧情或未公开跑团信息 | `【【剧透警告】】` |
| 非正史 | 不自动进入主设定事实 | `【【非正史】】` |
| 跑团记录 | 来源于具体战役或游戏记录 | `【【跑团记录】】` |
| 废案归档 | 保留参考价值，但不作为正式设定 | `【【废案归档】】` |
| DND3.5E | 涉及 DND 3.5E 规则内容 | `【【DND3.5E】】` |
| DND5.5E | 涉及 DND 5.5E 规则内容 | `【【DND5R】】` |
| 通用规则 | 不绑定单一规则版本 | `【【通用规则】】` |

### 单一页面多版本模板

```
【【规则双版本|默认=DND3.5E】】
这里写两个版本都共用的种族描述。
【版本开始|DND3.5E】
这里写3.5E内容
【版本结束】
【版本开始|DND5.5E】
这里写 5.5E 内容。
【版本结束】
```

## 特殊语法与组件

### 特殊引用框

为保持页面风格统一，建议使用本站配置的引用框样式。

标准引用：

```html
<blockquote class="custom-blockquote-default">
内容
</blockquote>
```

提示引用：

```html
<blockquote class="custom-blockquote-tip">
内容
</blockquote>
```

注意引用：

```html
<blockquote class="custom-blockquote-warning">
内容
</blockquote>
```

### 引用块

引用块用于表现题词、世界内文献、人物引言、史书摘录或章节开头的风格化文本。

基本引用：

`{{Quote|引用内容|来源}}`

带补充说明：

`{{Quote|引用内容|来源|补充说明}}`

带样式参数：

`{{Quote|引用内容|来源|补充说明|style=;}}`

一般情况下，不建议使用 `style=` 参数。引用块样式已由全站 CSS 统一控制，只有在页面具有明确视觉需求时，才应使用少量内联样式。

### 剧透与隐藏文字

如页面涉及尚未公开的跑团内容、关键谜底、人物结局或重大剧情反转，应先使用**剧透警告**页面模板，再视情况隐藏具体文本。

语法：

`普通文本之后是((这是一段隐藏内容))。`

示例：

普通文本之后是((这是一段隐藏内容))。

该语法适用于短句或少量关键词，不适合隐藏大段正文。若需要隐藏整段或整节内容，应考虑拆分为单独页面，或在标题处明确标注剧透。

### 折叠区块

折叠区块适合用于较长的资料列表、可选阅读内容、跑团记录分幕、规则补充或多版本文本。

基本格式：

```html
<div class="faccordion">
  <h3>第一章</h3>
  <p>这里是第一章内容。</p>

  <h3>第二章</h3>
  <p>这里是第二章内容。</p>
</div>
```

折叠区块内的标题会自动成为可点击项目。请勿滥用折叠区块；若正文信息本身就是页面主干，应直接展示。只有在内容较长、属于可选阅读，或会打断主线阅读时，才建议折叠。

## 资产管理

图片、地图、图标等资产应优先上传至本维基，尽量避免直接引用外部链接。外部图片可能因网络波动、权限变化、防盗链或原站删除而失效，不适合作为长期资料库的稳定资源。

专属于单个页面的资产，建议存放于与该页面同路径的资产文件夹内。例如：页面 `/弥恩诸世/人物汇总/某人` 的肖像可存为 `/弥恩诸世/人物汇总/某人/portrait.png`。

多页面共用的公共资产，应上传至统一资产目录，并按类型建立子目录。例如：

- `/asset/symbols/logo.png`
- `/asset/maps/mien-world-map.png`
- `/asset/icons/status-anchor.png`

资产文件名应尽量使用英文小写字母、数字和连字符。不要使用空格、特殊符号或含义不明的随机文件名。

推荐命名：

- `meen-world-map.png`
- `north-desert-border.png`
- `aleseya-symbol.png`

上传后请尽量不要随意重命名资产文件。若确需重命名，应同步检查所有引用该资产的页面，避免图片失效。

## 版本、署名与来源

页面历史会记录编辑行为，但重要页面仍建议在正文末尾或页面讨论中说明主要贡献者、整理者与资料来源。多人协作页面尤其应保留贡献记录，以便后续修订时追溯文本来源。

示例：

- `2026-04-24：罗莎 创建页面初稿。`
- `本页面部分内容整理自《漠北远征》跑团记录。`

若页面内容来自他人创作、跑团记录、旧版文档或外部资料，应确认授权情况，并在页面中注明来源。


## 反馈与维护

本指南旨在维护维基文本的结构性与一致性，并不限制合理的创作表达。若您认为现有格式无法满足某类页面的需要，可以提出补充建议。

涉及页面结构、模板语法、全站 CSS 或脚本功能的修改，应先在项目协作群中讨论，再由管理者统一调整。
